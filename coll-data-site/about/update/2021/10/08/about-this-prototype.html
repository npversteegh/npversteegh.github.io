About this prototype

Oct 8, 2021

This prototype web site is intended to demo Jekyll as a way to publish Ohio University Libraries collections-as-data. Jekyll is a static site generator that is also built in to GitHub, making it easy to deploy on their free web hosting for open source projects.

Similar collections-as-data sites from other Universities include:

  • Collections as Data at Pitt, which uses Jekyll and github.io as documentation for a GitHub repository.
  • Data Foundry from the National Library of Scotland, which uses WordPress as a front end for Amazon S3.

Other library collections-as-data projects currently exist only as a GitHub repo:

  • Marriott Library

Using a static site generator like Jekyll enables significant integration, automation, and customization, but requires some skill and configuration to deploy. There are dozens of static site generators with a variety of pros and cons, including Sphinx which is popular for documentation in the Python community and works with Read the Docs, another host for open source documentation.

The LibStatic methodology statement from the University of Idaho makes the case for using Jekyll and the JAM stack for library projects. They use Jekyll as the engine for their CollectionBuilder and Oral History as Data projects. Tech-for-humanists tutorials site Programming Historian offers a tutorial on building a static website with Jekyll and GitHub pages which they recommend if you want an “entirely free, easy-to-maintain, preservation-friendly, secure website over which you have full control”.
