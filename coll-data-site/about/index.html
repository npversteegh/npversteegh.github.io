About

This website is a prototype for offering open-source data from Ohio University Libraries’ collections in standardized packages for research. Each collection has its own web page with important collection information, links to data in source, normalized, and extended formats, and a data dictionary that defines each element in the data set.

Many of these collections are held at Ohio University Libraries’ Mahn Center for Archives and Special Collections and have been digitized by Ohio University Libraries’ Digital Initiatives department and are available through our Digital Archives page.

Data Dictionary Dictionary

Each collection data set is described by a data dictionary. Each data dictionary documents the following aspects of the provided data elements:

Element Name
The element name is the name it appears under in the offered data sets. It is the column name in TSVs, and key string in JSON files.
CONTENTdm field nick
The nickname (system address) for the field used internally by CONTENTdm. Nearly always a string of six or fewer alphabetic characters.
CONTENTdm system property
If Yes, the element is a system property generated by CONTENTdm.
Required
If Yes, the element is required for all items in a CONTENTdm collection.
DC Mapping
The Dublin Core DCMI element mapping assigned to the field in CONTENTdm. Used in CONTENTdm for OAI-PMH harvesting.
Normalized
If Yes, this element has been edited for inclusion in the normalized data.
Description
A prose text description of the contents of the data element.
Encoding
The standard or format used to encode the metadata element.
Notes
Relevant information not covered by the above fields.
Examples
A list of examples drawn from the collection data.
